# Environment Variables Complete Reference

## Database Configuration

### Local Development
```properties
DB_URL=jdbc:mysql://localhost:3306/chatconnect
DB_USERNAME=root
DB_PASSWORD=More@2525
```

### Railway Production (AUTO-SET by MySQL Plugin)
```properties
DB_URL=jdbc:mysql://${MYSQL_HOST}:${MYSQL_PORT}/${MYSQL_DB}
DB_USERNAME=${MYSQL_USER}
DB_PASSWORD=${MYSQL_PASSWORD}
```

**What Railway Sets:**
- `MYSQL_HOST=mysql.railway.internal`
- `MYSQL_PORT=3306`
- `MYSQL_DB=chatconnect` (or your db name)
- `MYSQL_USER=root`
- `MYSQL_PASSWORD=<random-generated>`

## Application Configuration

### JWT Settings
```properties
# Change this to a strong random string (32+ characters)
JWT_SECRET=your-strong-secret-key-here-minimum-32-chars
JWT_EXPIRATION=86400000  # 24 hours in milliseconds
```

**Generate Secure Secret:**
```bash
# Option 1: openssl (Mac/Linux)
openssl rand -base64 32

# Option 2: PowerShell (Windows)
[Convert]::ToBase64String((1..32 | ForEach-Object {[byte](Get-Random -Min 0 -Max 256)}))

# Option 3: Online tool
# https://generate-random.org/base64-generator (pick 32+ chars)
```

### CORS & WebSocket
```properties
# Local Development
APP_CORS_ALLOWED_ORIGINS=http://localhost:5173
APP_WS_ALLOWED_ORIGINS=http://localhost:5173

# Production (Railway)
APP_CORS_ALLOWED_ORIGINS=https://your-frontend-domain.com
APP_WS_ALLOWED_ORIGINS=https://your-frontend-domain.com
```

**Notes:**
- Multiple origins: `https://domain1.com,https://domain2.com`
- No trailing slash
- Must match your frontend deployment URL

### JPA & Hibernate Settings
```properties
# Auto-create database tables
JPA_DDL_AUTO=update        # Development/Staging
JPA_DDL_AUTO=validate      # Production

# SQL logging
JPA_SHOW_SQL=true          # Development (verbose)
JPA_SHOW_SQL=false         # Production (quiet)

# Performance settings
spring.jpa.properties.hibernate.jdbc.batch_size=20
spring.jpa.properties.hibernate.order_inserts=true
```

### Firebase Configuration
```properties
# Path to Firebase service account key
FIREBASE_CONFIG_PATH=/path/to/firebase-key.json
# Or base64 encoded:
FIREBASE_KEY_BASE64=<base64-encoded-key>
```

### Server Configuration
```properties
# Railway automatically sets PORT from environment
# You don't need to set this manually
PORT=8080

# Graceful shutdown (production)
server.shutdown=graceful
server.shutdown.wait-for-requests-on-shutdown=true
```

## How Railway Sets Variables

### Option 1: Manual Entry (Easiest)
1. Railway Dashboard
2. Select your Spring Boot service
3. Go to "Variables" tab
4. Add each variable one by one
5. Service auto-restarts with new variables

### Option 2: .env File (Not Recommended)
Railway reads from .env but recommends using dashboard.

### Option 3: GitHub Secrets + Railway Integration
Use GitHub Actions to push variables to Railway:
```yaml
# .github/workflows/deploy.yml
- name: Deploy to Railway
  run: |
    railway up
  env:
    JWT_SECRET: ${{ secrets.JWT_SECRET }}
```

## Variable Priority (What Gets Used)

1. **Environment Variable** (Highest)
2. **System Property** (-Dproperty=value)
3. **application.properties default value** (Lowest)

Example:
```properties
# In application.properties
jwt.secret=${JWT_SECRET:default-value}

# If JWT_SECRET env var exists → use that
# If JWT_SECRET env var NOT set → use "default-value"
```

## Complete .env.production Example

```bash
# Database (set by Railway MySQL plugin automatically)
# Don't set these manually - Railway does it
# DB_URL=jdbc:mysql://mysql.railway.internal:3306/chatconnect
# DB_USERNAME=root
# DB_PASSWORD=<auto-generated>

# JPA Configuration
JPA_DDL_AUTO=update
JPA_SHOW_SQL=false

# JWT Secret - CRITICAL: Change this to a strong random value!
JWT_SECRET=your-strong-32-character-secret-key-goes-here
JWT_EXPIRATION=86400000

# CORS - Update with your deployed frontend URL
APP_CORS_ALLOWED_ORIGINS=https://your-frontend.vercel.app
APP_WS_ALLOWED_ORIGINS=https://your-frontend.vercel.app

# Firebase
FIREBASE_CONFIG_PATH=/tmp/firebase-key.json

# Spring Profile
SPRING_PROFILES_ACTIVE=prod

# Server
server.port=8080
```

## Railway Dashboard Variable Setup

### Step 1: Open Variables
1. Railway Dashboard → Select Spring Boot Service
2. Click "Variables" tab
3. Click "Add Variable"

### Step 2: Add Each Variable
```
Name: JPA_DDL_AUTO
Value: update

Name: JPA_SHOW_SQL
Value: false

Name: JWT_SECRET
Value: <your-generated-32-char-string>

Name: JWT_EXPIRATION
Value: 86400000

Name: APP_CORS_ALLOWED_ORIGINS
Value: https://your-frontend-url.com

Name: APP_WS_ALLOWED_ORIGINS
Value: https://your-frontend-url.com

Name: SPRING_PROFILES_ACTIVE
Value: prod
```

### Step 3: Verify MySQL Variables
Click "MySQL plugin" → "Variables" → Verify Railway created:
- MYSQL_HOST ✓
- MYSQL_PORT ✓
- MYSQL_DB ✓
- MYSQL_USER ✓
- MYSQL_PASSWORD ✓

### Step 4: Restart Service
Service auto-restarts after variable changes.

## Secrets You Should NEVER Commit

**Never push these to GitHub:**
- `JWT_SECRET` (use a placeholder)
- `DB_PASSWORD` (use placeholder)
- `FIREBASE_CONFIG_PATH` (or base64 key)
- Any API keys, tokens, or credentials

**Instead:**
1. Add to .gitignore:
   ```
   .env
   .env.*.local
   firebase-key.json
   ```

2. Use Railway dashboard to set actual values

3. Use environment variables in code:
   ```java
   @Value("${JWT_SECRET}")
   private String jwtSecret;
   ```

## Troubleshooting Variables

### Problem: "Variable not being used"
- ✓ Check variable name is case-correct
- ✓ Check syntax in application.properties uses `${VARIABLE_NAME}`
- ✓ Restart service after changing variable
- ✓ Check Railway dashboard shows the variable

### Problem: "Default value always used"
- ✓ Environment variable not set or spelled wrong
- ✓ Check Railway Variables tab
- ✓ May need 60 seconds for Railway to apply

### Problem: "SQL errors"
- ✓ Check DB_URL, DB_USERNAME, DB_PASSWORD
- ✓ Verify MySQL plugin is added to Railway
- ✓ Check MySQL plugin shows connection string
- ✓ Try restarting service

## Quick Checklist

- [ ] JWT_SECRET is 32+ random characters
- [ ] APP_CORS_ALLOWED_ORIGINS matches frontend URL
- [ ] No trailing slashes in URLs
- [ ] JPA_DDL_AUTO is set (update for dev, validate for prod)
- [ ] MySQL plugin added to Railway
- [ ] All variables set in Railway dashboard
- [ ] Service restarted after variables added
- [ ] Logs show "Application started successfully"
